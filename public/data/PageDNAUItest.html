UI Implementation Test:

The purpose of this test is to construct an HTML table with two columns, representing the states: 'A', and 'B'. In each row there are four colored boxes, which may be moved back and forth between the two columns by clicking on them. Each row has a specific relative sequence the colors must retain as boxes are moved between columns.

Initial Data

You have been provided with a json-formatted data file (rows.json) for this test. In your test application, you must access this resource by way of an ajax call, and use it to initialize your user interface. The data in the file is an array of object literals, each representing a row for your table. Each object literal has two properties:
"sequence" -> defines the required relative sequence of colors for that row and "states" -> specifies the initial state for each color in that row, i.e. whether it starts in the 'A' or 'B' column on page load

Moving Boxes

If your table has one non-header row, and starts like:

AB
[b][y][o][r] 
You should be able to click [y] and [r], and end up with a row that looks like:

AB
[b][o][y][r]
Note that each cell in the row will retain the original relative sequence: in this example: "b", "y", "o", "r", i.e. [b] comes before [o], and [y] comes before [r].

Clicking on [y] and [r] again will move them back to column 'A', restoring you to the original state.

Column Count

Your UI should maintain a running count of how many colored boxes there are in each column in the headers for each column. The following example has three rows:
A (5)B (7)
[b][y][r][o]
[y][r][b][o]
[b][o][y][r]
Clicking on [o] in row 2, column 'B' will move [o] to column 'A' and update counts in the headers:

A (6)B (6)
[b][y][r][o]
[y][r][o][b]
[b][o][y][r]

Creating new columns

The user should be able to enter new state names into an input field and submit them, after which they are added to the table as a new column. Clicking on the colored boxes will move the box successively from left to right through the new columns. When they reach the right-most column, another click moves them back to the left-most column (A).

Rules for new column names:

  1. no duplicates of existing column names
  2. first character must be a capital letter between A-F or N-Z
  3. the leading character may (but is not required to) be followed by a number up to four digits long, which must be separated from the leading character by an underscore or dash if used
OK: "C" "Z" "N-1234"
BAD: "c" "I" "C1234"

Example table: adding state "N-1234" and clicking [o] to move it to the new column
A (6)B (5)N-1234 (5)
[b][y][r][o]
[y][r][o][b] 
[b][o][y][r] 

Regenerating rows.json

Finally, there should be a clickable button/link, which will cause rows.json to be regenerated based on the current state of the table. The updated json can be displayed in a modal or in a text area.
